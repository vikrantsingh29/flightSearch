# Flight Search Tracker

GitHub Pages URL:
[https://vikrantsingh29.github.io/flightSearch/](https://vikrantsingh29.github.io/flightSearch/)

This app tracks BLR -> FRA options with:
- Emirates and Qatar Airways from official-source scraper output
- Gulf Air and Air India via official fallback links

## Modes

Local live mode:
- Run `npm start`
- Open `http://127.0.0.1:8787/flight-tracker-fixed.html`
- This queries the local scraper against the official Emirates and Qatar Airways public sources in real time

GitHub Pages mode:
- Open [https://vikrantsingh29.github.io/flightSearch/](https://vikrantsingh29.github.io/flightSearch/)
- This serves published snapshot JSON generated by GitHub Actions from the same scraper
- Future Qatar dates also include a direct booking-fares link when public status is unavailable

## Deployment

GitHub Pages deployment is handled by `.github/workflows/deploy-pages.yml`.

What it does:
- builds a static Pages bundle into `dist/`
- runs the local scraper during the build
- publishes snapshot JSON for the last 3 days plus today and the next 14 days
- deploys the bundle to GitHub Pages on pushes to `master`, manual workflow runs, and a scheduled refresh

## Build Pages Locally

Run:

```bash
npm run build:pages
```

This creates a `dist/` folder with:
- `index.html`
- `flight-tracker-fixed.html`
- `tracker-app.js`
- `data/*.json`
- `data/manifest.json`

## Notes

- GitHub Pages is static, so it cannot run the live Node helper directly.
- The deployed site therefore shows published snapshots, not direct live server-side scraping from the browser.
- When an airline does not publish a usable public result for a date, the app falls back to official airline and airport links instead of fake data.
